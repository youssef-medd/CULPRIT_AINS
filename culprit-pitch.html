The Problem

Enterprise agents are
non-deterministic.

The same JSM ticket can produce different tool calls and different routing on two runs. This breaks every safety net.

Pass/Fail Tests

Don't apply — there is no exact output to match against. The agent takes a different valid path each time.

Silent Failures

All tools return "ok", no error fires, and the ticket is still misrouted. You find out after the damage.

Useless Signal

"Task failed" tells an engineer nothing about which step to fix: retrieval? planning? tool params? synthesis?

● Inside Atlassian: no mechanism exists today to detect that an agent has drifted from spec, or to attribute a failed run to a component.

Engineering Depth · 50%

The attribution algorithm.

Earliest high-confidence violation = the decisive suspect. Counterfactual replay turns correlation into causation.

# The core algorithm (simplified)
def attribute(trajectory, verdicts, tau):
    if end_to_end(verdicts) == "pass":
        return Attribution(verdict="pass")

    # earliest high-confidence violation = decisive suspect
    suspects = [v for v in component_verdicts(verdicts)
            if v.verdict == "fail" and v.confidence >= tau]
    suspects.sort(by=step_index)

    for s in suspects:
        edits = propose_minimal_edits(s, k=4)
        for e in sorted(edits, by=edit_size):
            if replay_from(trajectory, s.step_id, edit=e) == "task_succeeded":
                return Attribution(decisive=s, confirmed=True, repair=e, crs=...)

Earliest-violation rule

Stops Culprit from blaming a downstream symptom when an upstream step poisoned its input.

Counterfactual confirmation

Fixing the suspect must flip the outcome. If it does → causally confirmed. If not → reported as unconfirmed.

Two-Layer Evaluation

Two layers.
Complementary, not redundant.

A deterministic state machine catches structural faults instantly. LLM judges catch what no state machine can express.

Shadow Monitor

Deterministic

Each contract's ordering invariants compile into a state machine that runs online, in parallel with the agent.

● Wrong tool order

● Missing required step

● Incapable tool selected

● Field accessed before retrieval

→ Mid-flight divergence alert before any judge runs

LLM Judges

Semantic

Per-step LLM judges score each step against its rubric with focused context and self-consistency.

● Was retrieved context relevant?

● Is the summary grounded?

● Did the plan use the right signals?

● Was the priority correctly derived?

→ k-sampled → agreement = confidence → debate on splits

Checks action sequences, not exact text — legitimately different valid paths don't trip it

How ground truth works

The label comes from the injection, not from an LLM oracle. A trajectory mutation engine programmatically corrupts known-good runs: semantic noise, truncated results, reordered retrieval, parameter perturbation.

What we measure

Attribution accuracy, per-category precision / recall / F1, step-localization accuracy, and counterfactual confirmation rate — benchmarked against the published ~14.2% SOTA.

These numbers prove the machinery is correct end-to-end — not that LLM judges hit this accuracy in the wild

Non-Determinism

Non-determinism,
handled on two timescales.

The agent is non-deterministic by design. Culprit's evaluation is robust to it.

WITHIN A RUN

Self-Consistency + Debate

● k-sample each judge → agreement = confidence

● Genuine disagreement → pass-vs-fail debate

● Senior Judge adjudicates → still unresolved → human

● Shadow Monitor checks action sequences, not text

ACROSS RUN BATCHES

Drift Detection

● PSI / KL divergence across run batches

● Flags behavioral distribution shifts over time

● Complementary to within-run divergence alerts

● Two timescales: online (instant) + batch (trend)

Key insight: Different valid paths don't trip the monitor — invariants are over action sequences, not exact text. A legitimately different routing that still respects ordering constraints passes cleanly.

14 / 15
Honest Limitations

We trade breadth
for measurable accuracy.

General-purpose trace debugging sits at ~11–14%. Culprit wins by not being general — and we report the risks honestly.

⚠

Counterfactual replay

Highest-risk component. Degrades gracefully to correlation-based attribution if no edit flips the outcome. The E3 Must still holds.

⚠

Meta-eval numbers

Produced by the deterministic heuristic backend over a small by-construction-labeled set. They prove the machinery, not wild-LLM accuracy.

⊙

Deliberate scoping

One JSM triage agent, fixed step types, explicit per-step contracts. General trace debugging is a frontier problem at ~11%; we win by narrowing.

✓

Degraded modes

Judge timeout → retry once → emit unknown at confidence 0. Missing fields treated as signal, not crash. Failed replay → correlation fallback.

Run with a key and a noisier corpus for the realistic LLM-judge regime
